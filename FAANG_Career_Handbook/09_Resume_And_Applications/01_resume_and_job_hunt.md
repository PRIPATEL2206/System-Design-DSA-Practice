# Resume & Job Applications — Get Interviews at Top Companies

## The Resume That Gets Callbacks

### The Truth About Resume Screening
- Recruiter spends 6-10 seconds on first pass
- ATS (Applicant Tracking System) filters by keywords BEFORE human eyes
- Your resume must pass both AI and human review

### The Ideal Format

```
┌─────────────────────────────────────────────────────┐
│  PRINCE PATEL                                        │
│  System Engineer | Python • AWS • PySpark • ML       │
│  📧 email │ 📱 phone │ 🔗 LinkedIn │ 💻 GitHub     │
├─────────────────────────────────────────────────────┤
│  PROFESSIONAL SUMMARY (2-3 lines)                    │
│  [Role] with [X years] expertise in [core skills].  │
│  Built [impressive thing]. Seeking [target role].    │
├─────────────────────────────────────────────────────┤
│  EXPERIENCE                                          │
│  [Company] — [Title]               [Date range]     │
│  • [Action verb] + [what you did] + [impact/metric] │
│  • [Action verb] + [what you did] + [impact/metric] │
│  • [Action verb] + [what you did] + [impact/metric] │
├─────────────────────────────────────────────────────┤
│  PROJECTS (2-3 strongest)                            │
│  [Project Name] — [One line description]             │
│  • Tech stack + key achievement + link               │
├─────────────────────────────────────────────────────┤
│  SKILLS                                              │
│  Languages: Python, SQL, Java                        │

├─────────────────────────────────────────────────────┤
│  EDUCATION                                           │
│  [Degree] — [University]           [Year]           │
├─────────────────────────────────────────────────────┤
│  CERTIFICATIONS                                      │
│  AWS Solutions Architect – Associate                 │
└─────────────────────────────────────────────────────┘
```

### Rules
- **1 page maximum** (unless 10+ years experience)
- **No photos** (bias, ATS issues)
- **No "Objective" section** (use Summary instead)
- **No generic skills** like "Microsoft Office" or "teamwork"
- **Consistent formatting** (same font, alignment, date format)
- **PDF format** (preserves formatting across systems)

## The Bullet Point Formula: XYZ

Every bullet point should follow:

> **Accomplished [X] as measured by [Y], by doing [Z]**

### Examples

**Bad (task-focused, no impact):**
- Worked on ETL pipelines
- Used PySpark for data processing
- Maintained AWS infrastructure

**Good (impact-focused, with metrics):**
- Reduced ETL pipeline runtime by 65% (from 4 hours to 1.4 hours) by optimizing PySpark job partitioning and implementing predicate pushdown on 500M+ row datasets
- Built real-time anomaly detection system processing 100K events/minute using AWS Lambda and Kinesis, reducing fraud detection time from 24 hours to under 30 seconds
- Designed and deployed data warehouse on Redshift serving 50+ analysts, cutting average query time from 45s to 3s through strategic materialized views and sort key optimization

### Power Verbs for Engineers
| Category | Verbs |
|----------|-------|
| Built/Created | Architected, Designed, Developed, Engineered, Implemented |
| Improved | Optimized, Reduced, Accelerated, Streamlined, Enhanced |
| Led | Spearheaded, Drove, Led, Mentored, Championed |
| Delivered | Shipped, Deployed, Launched, Released, Migrated |
| Solved | Debugged, Resolved, Eliminated, Mitigated, Automated |

### Quantifying Impact When You Don't Have Numbers
| Situation | How to Quantify |
|-----------|----------------|
| "Made it faster" | Measure before/after: "Reduced from X to Y (Z% improvement)" |
| "Handled data" | Volume: "Processing 500M records daily" |
| "Served users" | Scale: "Serving 10K daily active users" |
| "Improved reliability" | Uptime: "Achieved 99.9% availability" |
| "Saved effort" | Time: "Automated process saving 20 hours/week" |
| No metrics? | Use estimates: "Reduced by ~40%" (indicate approximate) |

## Tailoring Your Resume for Each Application

### The 80/20 Rule
Keep 80% of your resume constant. Customize 20% for each job:
1. **Summary line** — Match their role title and key requirements
2. **Skill ordering** — Put their required skills first
3. **Bullet point emphasis** — Lead with most relevant achievements
4. **Keywords** — Mirror exact words from the job description

### Example Customization
**Job posting says:** "Experience with distributed systems, Python, and ML pipelines"

**Your bullet becomes:**
> "Designed distributed data processing pipeline in Python handling 100M events/day with ML-based anomaly detection, deployed across 3 AWS regions"

(Hits: distributed, Python, ML pipelines — exact keywords)

## Job Application Strategy

### The Funnel
```
Applications Sent:     100
Recruiter Screens:     15-20  (15-20% response rate with good resume)
Technical Screens:     10-12  (60-80% pass recruiter)
Onsite/Final Round:    5-7    (50-60% pass tech screen)
Offers:                2-3    (40-50% pass onsite)
```

### Where to Apply (Priority Order)

**Tier 1: Referrals (3-5x higher response rate)**
- Ask connections at target companies
- LinkedIn message template:

```
"Hi [Name], I'm a [role] at [company] with experience in [their tech stack]. 
I'm really interested in the [specific role] at [their company] because [specific reason]. 
Would you be open to referring me? I'd be happy to share my resume. 
Either way, would love to connect!"
```

**Tier 2: Direct Applications (with strategy)**
- Apply within 48 hours of job posting (early applicants get priority)
- Target companies where you know the tech stack
- Apply to 3-5 roles per company (different teams, same level)

**Tier 3: Recruiters on LinkedIn**
- Set profile to "Open to Work" (visible to recruiters only)
- Respond to inbound messages promptly
- Be specific about what you want: role, level, tech stack, location

### LinkedIn Optimization
```
Headline: System Engineer @ TCS | Python • AWS • PySpark • ML | Building Data Pipelines at Scale

About: 
[2-3 sentences about what you do and your impact]
[1 sentence about what you're passionate about]
[1 sentence about what you're looking for]

Skills to add (in order of relevance):
Python, AWS, PySpark, Apache Spark, SQL, ETL, Data Engineering,
Machine Learning, FastAPI, Docker, PostgreSQL, Redis, Kafka
```

## Application Timeline (12-Week Plan)

| Week | Action |
|------|--------|
| 1-2 | Polish resume, update LinkedIn, identify 30 target companies |
| 3-4 | Reach out for referrals (5/week), start applying (10/week) |
| 5-6 | Continue applying, follow up on sent referrals |
| 7-8 | First interviews arriving — practice intensively |
| 9-10 | Interview pipeline in full flow |
| 11-12 | Negotiate offers, make decision |

### Parallel Processing Strategy
- Apply to safety companies (where you're confident) AND stretch companies (FAANG)
- Use safety offers as leverage for stretch negotiations
- NEVER stop applying until you've signed an offer letter

## Salary Negotiation (Brief)

### Key Principles
1. **Never name a number first** — "I'm looking for a competitive offer aligned with the market for this role and level."
2. **Research market rates** — levels.fyi, Glassdoor, Blind
3. **Negotiate base, RSU, sign-on bonus, and level separately**
4. **Use competing offers** — "I have another offer at $X. Can you match/beat?"
5. **Be polite but firm** — "I'm very excited about this role. To accept, I'd need the compensation to be at [X]."
6. **Everything is negotiable** — Start date, remote days, learning budget, team choice

## Common Mistakes

1. **Applying without tailoring** — Generic resume → generic rejection
2. **Only applying online** — Referrals are 3-5x more effective
3. **Not following up** — Send a brief follow-up after 1 week
4. **Applying to wrong level** — Under-leveling is common for Indian engineers. Apply to the level you WANT to be, not where you are.
5. **Waiting to be "ready"** — You'll never feel ready. Start applying when you're at 70% preparation.
6. **Putting all eggs in one basket** — Apply broadly, interview is a skill that improves with practice
