## Golden Rules for DSA Interviews

1. **Clarify** — Ask about input size, constraints, edge cases
2. **Brute Force First** — State the obvious O(n²) approach
3. **Optimize** — Identify the pattern (sliding window, two pointer, etc.)
4. **Code** — Write clean, readable code with good variable names
5. **Test** — Walk through with an example, check edge cases
6. **Complexity** — State Time and Space complexity
