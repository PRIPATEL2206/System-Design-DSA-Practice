# Resume That Gets Interviews — Complete Guide

## The Truth About Resumes

```
Average time a recruiter spends on your resume: 6 SECONDS.
Your resume must: 
  1. Pass ATS (Applicant Tracking System) — keyword matching
  2. Catch recruiter's eye in 6 seconds — clear impact
  3. Give interviewer talking points — interesting projects
```

---

## 1. Format Rules (Non-Negotiable)

```
✅ DO:
  - Single page (ALWAYS, even with 10 years experience)
  - Clean, simple format (no graphics, colors, tables)
  - Consistent formatting (same font, bullet style)
  - PDF format (preserves layout)
  - Margins: 0.5-0.75 inches
  - Font: 10-11pt (Calibri, Arial, or similar)
  - Section order: Summary → Experience → Projects → Skills → Education
  
❌ DON'T:
  - Photos (bias concerns, especially US companies)
  - Objective statement (outdated, use summary instead)
  - "References available upon request" (waste of space)
  - Personal info beyond email/phone/LinkedIn/GitHub
  - Skills like "MS Office" or "Communication" (obvious)
  - Paragraphs (use bullet points ONLY)
  - More than 1 page
```

## 2. The Winning Formula for Bullet Points

### XYZ Formula (Google-recommended)
```
"Accomplished [X] as measured by [Y], by doing [Z]"

❌ Bad:  "Worked on data pipeline"
❌ Bad:  "Responsible for ETL processes"
✅ Good: "Reduced ETL pipeline runtime by 65% (from 4hrs to 1.4hrs) 
          by implementing PySpark partitioning and caching strategies, 
          processing 2TB daily data"

❌ Bad:  "Developed APIs"
✅ Good: "Built RESTful APIs serving 50K+ daily requests with 99.9% 
          uptime using FastAPI, reducing response latency by 40%"

❌ Bad:  "Worked with AWS services"
✅ Good: "Architected serverless data pipeline using AWS Glue + Lambda + S3, 
          eliminating $3K/month EC2 costs while improving reliability"
```

### Power Verbs (Start Every Bullet With These)
```
Engineering:    Built, Designed, Architected, Implemented, Optimized, 
                Migrated, Automated, Scaled, Reduced, Refactored

Leadership:     Led, Mentored, Coordinated, Spearheaded, Drove, 
                Initiated, Established, Championed

Impact:         Improved, Increased, Reduced, Eliminated, Accelerated,
                Saved, Generated, Delivered, Achieved

Analysis:       Analyzed, Identified, Diagnosed, Investigated,
                Evaluated, Discovered, Resolved

NEVER start with: Responsible for, Helped with, Assisted in, Worked on
```

## 4. ATS Optimization

```
ATS (Applicant Tracking System) scans your resume for keywords.
If keywords don't match job description → auto-rejected!

Strategy:
1. Read job description carefully
2. Identify key technical terms and requirements
3. Include those EXACT terms in your resume
4. Don't use abbreviations alone: "AWS (Amazon Web Services)"
5. Use standard section headings: "Experience", "Skills", "Education"

Job says: "Experience with Apache Spark and data lake architectures"
Your resume: "Built data lake on S3 using Apache Spark and Delta Lake format"
                                    ↑ keyword match! ↑

Pro tip: Run your resume through jobscan.co to check ATS match %
Target: 75%+ keyword match with job description
```

## 5. Tailoring Per Company

```
DON'T send the same resume to every company!

For each application:
1. Read the job description 3 times
2. Identify top 5 requirements
3. Reorder your bullets to match (most relevant first)
4. Add/modify project descriptions to match tech stack
5. Use their terminology (if they say "microservices", you say "microservices")

Example:
  Google JD mentions: "distributed systems, data processing at scale"
  → Lead with your PySpark 5TB pipeline bullet
  
  Amazon JD mentions: "customer impact, ownership"
  → Lead with the bullet showing customer-facing improvement
```

## 6. LinkedIn Profile Tips

```
- Headline: "System Engineer | Python | AWS | PySpark | Building ML Systems"
  (NOT just "System Engineer at TCS")

- About section: Tell your story (300 words max)
  "I build data pipelines that process terabytes daily and ML systems 
  that serve predictions at scale. Currently at TCS, working on..."

- Featured: Pin your best projects, blog posts, or GitHub repos

- Activity: Post about tech (1-2 posts/week), comment on others
  This gets you noticed by recruiters!

- Open to Work: Turn ON (visible to recruiters only)

- Skills: List 50 skills (more endorsements = higher in recruiter search)
```

---

## 7. GitHub Profile Tips

```
Your GitHub IS your portfolio. Make it count.

Must-have:
  - Profile README with introduction
  - 3-5 pinned repositories (your best work)
  - Each repo: Clear README, clean code, documentation
  - Contribution graph should be GREEN (shows consistency)
  - At least 1 project demonstrating system design skills
  - At least 1 ML/AI project

Each pinned repo README should have:
  - What it does (1-2 sentences)
  - Architecture diagram (simple text diagram)
  - How to run it
  - Tech stack used
  - Demo/screenshots if applicable
```

## 8. Cover Letter (When Required)

```
Keep it SHORT (150-200 words). Three paragraphs:

Para 1: Why this company specifically (show you researched them)
  "I'm excited about [Company]'s work on [specific project/product]. 
  As someone who has built [related work], I see a strong alignment..."

Para 2: Why you're a fit (connect your experience to their needs)
  "In my role at TCS, I [specific achievement relevant to this role]. 
  This experience directly maps to your need for [requirement from JD]."

Para 3: Call to action
  "I'd love to discuss how my experience with [X] and [Y] can 
  contribute to your team's goals around [Z]. Available for a 
  conversation at your convenience."

That's it. Short, specific, relevant.
```
